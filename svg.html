Inline SVG used for buttons and links

SVGs are currently still very problematic with all screen readers. Therefore I do not suggest using nested title and desc elements within an SVG element. Nor do I suggest assigning role="img" to an SVG.

Also, stay away from using aria-labeledby and aria-describedby. This causes repeating phrases.

Known issues:

Below are just some of the issues using svg[role="img"] and using nested title and desc elements.

Problems with JAWS:

  • JAWS will run the title and the word "icon" together as one word causing pronunciation issues.
  • JAWS announces a link with an SVG assigned a role="img" as a "link graphic". This does not add anything to the semantics of the link.

Problems with NVDA:

  • NVDA will read the title text multiple times when an SVG is nested in an anchor or a button

Problems with VoiceOver:

  • VoiceOver does not recognize the title or desc element.

Preferred method:

  • Assign the attribute role="presentation" to the SVG element.
  • Assign the attribute focusable="false" to the SVG element.
  • Use accessibly-hidden text to give the button or link meaning for those using a screen reader.

<button type="button">
  <span class="accessibly-hidden">Reveal navigation</span>
  <i class="svg">
    <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 31.8 25" role="presentation" focusable="false">
      <rect fill="#000000" width="31.8" height="5"></rect>
      <rect y="10" fill="#000000" width="31.8" height="5"></rect>
      <rect y="20" fill="#000000" width="31.8" height="5"></rect>
    </svg>
  </i>
</button>
