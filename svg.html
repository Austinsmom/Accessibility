What about using title and desc?

SVGs are currently still very problematic with many screen readers. Therefore I do not suggest using nested title and desc elements within an SVG element. Nor do I suggest using role="img", assuming you want to mimic a nested image tag.

If we consider a nested image tag as the gold standard to be mimicked by the svg tag we should stay away from the svg desc or title tags and, believe it or not, the role="img" attribute. Use aria-label for the button and screen-reader-text for the link. Use role="presentation" on the svg tag itself.

Nor do I suggest assigning role="img" to an SVG, assuming we want to mimic a button or link with a nested image tag.

Also, stay away from using aria-labeledby and aria-describedby. This causes repeating phrases.

The following findings are based on tests done in December 2016. Find the test codepen page to do your own research.

Known issues:

Below are just some of the issues when using nested title and desc elements as well as using role="img".

Problems with NVDA:

  • NVDA will read the title text multiple times when an SVG is nested in an anchor or a button

Problems with VoiceOver:

  • VoiceOver does not recognize the title or desc element.

Why use SVG instead background image

SVG is preferable for icons over background images/sprites because if PC is in High Contrast Mode, SVGs are still visible. Background images are not rendered.

SVGs can be scaled to their parent container (as long as you don't set width and height to SVG).

Preferred method:

  • Assign the attribute role="presentation" to the SVG element.
  • Assign the attribute focusable="false" to the SVG element.
  • Use hidden screen-reader-text to give the button or link meaning for those using a screen reader. See the "screen-reader-text" class in base.css.

Button example:


<button type="button" style="width: 50px; height: 50px;">
  <span class="screen-reader-text">Reveal navigation</span>
  <svg xmlns="http://www.w3.org/2000/svg" viewBox="00 0 90 90" role="presentation" focusable="false">
    <circle cx="45" cy="45" r="40" stroke="black" stroke-width="3" fill="red">
  </svg>
</button>

Link example:

search google 

<a href="http://www.google.com/" style="display: inline-block; width: 50px; height: 50px;">
  <span class="screen-reader-text">search google</span>
  <svg xmlns="http://www.w3.org/2000/svg" viewBox="00 0 90 90" role="presentation" focusable="false">
    <circle cx="45" cy="45" r="40" stroke="black" stroke-width="3" fill="blue">
  </svg>
</a>
