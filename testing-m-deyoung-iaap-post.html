Using images in place of radio buttons

Accessibility Tests

Requirements:

  • Do not use a mouse for this test.
  • It is best to test in JAWS+IE or NVDA+FF.
  • Test with keyboard only first.
  • Retest with keyboard + screen reader.

Pass:

Keyboard only

  • As I tab into the radio button cluster I can see a focus outline around the radio button with focus.
  • As I use my space bar or my arrows to check/uncheck a particular radio button the state change is visually obvious.

Keyboard + screen reader

  • As I tab into the radio button cluster I hear the legend describing what the radio buttons are for (JAWS/NVDA only).
  • As each radio button receives focus I hear the label and the checked state of the radio button.

Fail:

Keyboard only

  • When tabbing into a radio button cluster I do not see a focus outline around the radio button with focus.
  • As I use my space bar or arrow keys to check/uncheck a particular radio button I can't visually tell whether it is in a checked or unchecked state.

Keyboard + screen reader

  • I can't hear the legend and therefore do not know what this radio button cluster is for (JAWS/NVDA only).
  • I cannot hear the label and the checked state of the radio buttons announced when it receives focus.

Accessible Solution notes

  • Radio buttons and labels are not moved off-screen.
  • Radio buttons and labels are hidden behind icons.
  • Fieldset and legend is used to give radio cluster meaning.
  • Label wraps around input field and icon, making the whole thing clickable.
  • If a background image is used, the text behind will be seen in high-contrast mode.
  • If SVGs are used (as shown) they will still be visible in high contrast mode.
  • focus is placed around the icon using sibling selector.
  • checked state color is assigned using sibling selector.
